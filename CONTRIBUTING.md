# Contributing

This project is a manual MVP for AI repo re-entry / handoff audits.

Contributions are welcome when they improve:

- documentation clarity
- accident pattern wording
- re-entry maturity rank clarity
- intake form clarity
- sample output clarity
- public-surface boundary wording

## Not Open By Default

The following work is not open by default:

- implementation
- automation
- scraping/API work
- pricing or sell-test material
- outreach or posting
- release/tag work
- license/reuse selection

Those areas remain HOLD or BLOCK until an explicit project gate changes them.

## Contribution Guidelines

- Keep changes small and documentation-focused.
- Do not add secrets, credentials, private logs, personal data, or confidential company material.
- Do not submit code or tooling changes unless they are explicitly requested by a future project gate.
- Do not imply safety, legal, security, medical, or implementation approval.
- Preserve the manual nature of the MVP.
