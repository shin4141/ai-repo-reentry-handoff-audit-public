# Private Exclusion Rules

## Private-Only Exclusions

Do not copy these from the private/source B repo into this public shelf:

- `.git/`
- `handoff/`
- raw `audits/`
- target resolution traces
- self-audits with local paths
- Entry/Pain-specific internal audits unless rewritten as sanitized examples and explicitly approved
- `AGENTS.md` unless rewritten as public contributor guidance
- private repo history
- implementation files not intended for public readers
- pricing/sell-test material
- outreach/posting material

## Local Path / Operator Context Exclusion Rule

Remove or generalize local machine paths, private workspace paths, operator-only names, and internal decision notes unless explicitly approved for public wording.

## Audited Repo Details Exclusion Rule

Do not expose audited repo private details, target-resolution traces, discovery logs, or internal handoff context.

## Do-Not-Copy Rule

When in doubt, derive a clean public-facing document instead of copying the private/source file.
