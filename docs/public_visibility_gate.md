# Public Visibility Gate

This public shelf separates local public-candidate preparation from actual public exposure.

## Visibility Gate

Before changing repo visibility or creating a public repo, check:

- README top badges show the current status
- no private operator context
- no private repo paths
- no audited repo private details
- no accidental pricing / outreach / implementation claims
- API / scraping / automation remains BLOCK
- license / reuse status is explicit
- support / security / issue surfaces are intentional
- raw internal handoff and audit history are excluded or sanitized

## Announcement Gate

After any future visibility change, before posting or star-seeking, check:

- rendered README first screen
- About description, if applicable
- Topics, if applicable
- badges render correctly
- README hook is visible
- Accident Menu is visible
- Minimum Input Pack is visible
- Sample Output link works
- license/reuse status is visible
- no contradictory GO/HOLD/BLOCK wording

## Do Not Decide Here

This file does not decide visibility, announcement, license, pricing, release, implementation, API use, scraping, or automation.
